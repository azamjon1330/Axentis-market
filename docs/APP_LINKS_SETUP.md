# App Links: мгновенное открытие ссылок в приложении (необязательно)

## Что уже работает без этого

Ссылки `https://axentis.uz/product/123` и `/company/45` **уже** открывают
приложение (если установлено) или сайт — через промежуточную страницу
(~1 секунда). Ничего настраивать не обязательно.

## Что даст этот шаг

Android будет открывать приложение **мгновенно**, вообще без браузера —
как у Яндекс Маркета. Для этого Google должен убедиться, что сайт axentis.uz
и приложение принадлежат одному владельцу. Доказательство — SHA256-отпечаток
подписи приложения, опубликованный на сайте.

## Шаги (5 минут, нужен доступ к аккаунту Expo)

1. На своём компьютере в папке `Homepage` выполнить:

   ```bash
   npx eas credentials
   ```

   Выбрать **Android** → профиль **production**. В выводе найти строку
   **SHA256 Fingerprint** — длинная строка вида
   `A1:B2:C3:...` (32 пары символов через двоеточие).

   *Альтернатива:* Google Play Console → ваше приложение →
   **Setup → App signing** → скопировать «SHA-256 certificate fingerprint».

2. Открыть файл `public/.well-known/assetlinks.json` в этом репозитории и
   заменить строку-заглушку
   `ЗАМЕНИТЕ_НА_SHA256_ОТПЕЧАТОК_ПОДПИСИ_ПРИЛОЖЕНИЯ`
   на скопированный отпечаток (кавычки оставить).

3. Закоммитить, задеплоить сайт (файл должен открываться по адресу
   `https://axentis.uz/.well-known/assetlinks.json`).

4. Пересобрать приложение и выложить обновление:

   ```bash
   cd Homepage
   npx eas build --platform android
   ```

   (Перехват ссылок — `intentFilters` в `app.json` — уже добавлен в коде.)

После установки обновления Android сам проверит файл на сайте и начнёт
открывать ссылки axentis.uz прямо в приложении.

## iOS (когда появится сборка для App Store)

В файле `public/.well-known/apple-app-site-association` заменить `TEAMID`
на ваш Apple Team ID (Apple Developer → Membership). `associatedDomains`
в `app.json` уже настроен.
