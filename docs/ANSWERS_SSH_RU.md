## ❓ Вопрос 2: Почему загрузка фото НЕ работает на VPS но работает на localhost?

### 🔍 НАШЕЛ ПРИЧИНУ!

**На localhost (Docker):**
- ✅ Директория `./uploads` монтируется как Docker volume `uploads_data`
- ✅ Backend запущен внутри контейнера и видит эту директорию
- ✅ Nginx проксирует `/uploads` → `backend:3000`
- ✅ Файлы сохраняются и доступны

**На VPS (HTTP сервер):**
- ❌ Директория `./uploads` **НЕ СУЩЕСТВУЕТ** или **НЕТ ПРАВ** на запись!
- ❌ Backend запущен НЕ из корневой директории проекта
- ❌ Nginx на VPS **НЕ НАСТРОЕН** правильно для проксирования `/uploads`
- ❌ Путь к файлам сохраняется в БД, но файлы физически НЕ ЛЕЖАТ там!

### 📋 ЧТО НУЖНО ИСПРАВИТЬ НА VPS:

**1. Создать директорию uploads:**
```bash
mkdir -p /path/to/onlineshop2/uploads
chmod 755 /path/to/onlineshop2/uploads
```

**2. Убедиться что backend запускается из корня проекта:**
```bash
cd /path/to/onlineshop2/backend
./main  # НЕПРАВИЛЬНО - backend не видит ./uploads

cd /path/to/onlineshop2
./backend/main  # ПРАВИЛЬНО - backend видит ./uploads
```

**3. Настроить Nginx правильно:**

Если у вас на VPS есть Nginx (НЕ Docker), нужно добавить в конфигурацию:

```nginx
server {
    listen 80;
    server_name ваш_домен_или_ip;
    
    # Проксирование API
    location /api {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # ⭐ ВАЖНО! Проксирование uploads
    location /uploads {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        
        # ИЛИ если хотите раздавать напрямую:
        # alias /path/to/onlineshop2/uploads;
    }
    
    # Frontend
    location / {
        root /path/to/onlineshop2/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

## ❓ Вопрос 3: Почему миграции не выполняются автоматически на VPS?

### 🤔 ВОТ ПОЧЕМУ:

**На localhost (Docker):**
1. При запуске `docker compose up` создается НОВАЯ пустая база данных
2. Backend запускается и выполняет код в `database/database.go`
3. Функция `Migrate()` выполняет ВСЕ миграции автоматически
4. ✅ Таблицы создаются

**На VPS:**
1. База данных `onlineshop2` **УЖЕ СУЩЕСТВУЕТ**
2. Backend запускается и пытается выполнить миграции
3. НО! Если таблицы уже есть (частично), некоторые миграции пропускаются
4. Если база данных называется `azaton` (старое название), а код ищет `onlineshop2` - миграции НЕ выполняются вообще!

### 📝 ПОЧЕМУ АВТОМАТИКА НЕ РАБОТАЕТ:

```go
// backend/database/database.go
func Migrate(db *sql.DB) error {
    // Эта функция выполняет SQL миграции
    // НО! Если таблица УЖЕ существует - SQL вернет ошибку
    // Поэтому используется: CREATE TABLE IF NOT EXISTS
    
    // ПРОБЛЕМА: Если вы переименовали базу с 'azaton' → 'onlineshop2',
    // но в config.go стояло 'azaton' как дефолт,
    // backend подключался к СТАРОЙ базе!
}
```

**ПОЭТОМУ:**
- На VPS нужно **ВРУЧНУЮ** убедиться что:
  1. ✅ База данных называется `onlineshop2`
  2. ✅ Backend подключается именно к ней
  3. ✅ Все миграции выполнены

**Я МОГУ ЭТО СДЕЛАТЬ ЧЕРЕЗ SSH!** Просто дайте мне доступ.

**2. Неправильная директория**
- Команды выполнялись НЕ из корня проекта
- Пути к файлам были неправильные

**3. Недостаточно прав**
- Команды требовали `sudo`, но я не мог его использовать
- Нужно было подключиться как `root`

**4. База данных была заблокирована**
- PostgreSQL был запущен, но база была занята
- Нужно было остановить backend перед выполнением миграций

**5. Я не знал точную конфигурацию вашего сервера**
- Docker или нет?
- Какой путь к проекту?
- Какой пользователь PostgreSQL?
